# 贡献指南 (Contributing Guide)

感谢您对本项目的关注！欢迎通过以下方式贡献：

## 📝 贡献方式

### 1. 提交 Issue（通过 Atomgit Issues）
- **报告错误**：技能文件中的技术错误、错别字、链接失效
- **功能建议**：新增技能方向、改进现有内容
- **内容讨论**：对某个技术点的讨论和补充

### 2. 提交 Pull Request
- **修复错误**：修正技术描述、补充缺失内容
- **新增技能**：按现有模板格式新增技能文件
- **改进文档**：优化翻译、补充示例、增加配图

## 📋 技能文件模板

每个技能文件应遵循以下标准结构，包含 YAML frontmatter 和固定段落：

```markdown
---
id: "NN-NNN"                              # 模块ID-技能序号 (如 15-001)
title: "技能名称 (English Name)"          # 技能标题
category: "模块中文名"                     # 中文分类
category_en: "Module English Name"        # 英文分类
difficulty: "★★★★"                        # 难度（★~★★★★★）
tools: "工具1, 工具2, 工具3"              # 推荐工具
last_updated: "2025-07"                   # 最后更新
---

# 技能名称 (English Name)

## 概述
简要说明该技能的背景、适用场景和核心价值。

## 核心技能
技术原理、命令示例、代码片段。

## 常用工具

| 工具 | 用途 | 链接 |
|:---|:---|:---|
| Tool1 | 用途说明 | https://... |
| Tool2 | 用途说明 | https://... |

## 参考资源
- [资源名称](https://...)
- [资源名称](https://...)
```

> **重要**: 新增或修改技能后，请运行 `python skill_query.py validate` 确认 index.json 与实际文件一致。

## 📂 目录命名规范

- 模块目录：`{编号}-{中文名称}-{EnglishName}`
- 技能文件：`{中文名称}-{EnglishName}.md`

## ✅ 检查清单

提交 PR 前请确认：

- [ ] 内容准确，技术细节无误
- [ ] 遵循现有 Markdown 格式
- [ ] 链接有效（相对路径）
- [ ] 无敏感信息（IP、密码、密钥等）
- [ ] 文件编码为 UTF-8

## 🔒 安全声明

所有内容仅用于**授权的安全测试与合法的安全教育**。请勿用于非法用途。

## 📬 联系方式

如有问题，请通过 Atomgit Issue 进行讨论。
